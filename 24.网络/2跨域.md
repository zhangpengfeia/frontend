# 跨域问题及解决方案
## CORS
1.只要服务器明确表示允许，则校验通过
2.服务器明确拒绝或没有表示，则校验不通过
所以服务器必须为自己的服务器

cors将请求分为两类：简单请求和预检请求
只要满足以下条件，就是简单请求,否则就是预检请求：
1.请求方法是GET,POST,HEAD
2.请求头中没有自定义的请求头，只有Accept,Accept-Language,Content-Type,Content-Language,User-Agent
3.content-type 为 application/x-www-form-urlencoded,multipart/form-data,text/plain

### 跨域 cookie 问题
```js
// xhr 
const xhr = new XMLHttpRequest();
xhr.open('GET', 'http://localhost:5000/index.html');
xhr.withCredentials = true;

// featch api
fetch('http://localhost:5000/index.html', {
  credentials: true,
});
```
这样请求会携带cookie，但是服务器必须明确表示允许：Access-Control-Allow-Credentials: true
### 跨域获取响应头
服务器需要手动声明响应头，才能获取到响应头：Access-Control-Expose-Headers: 需要暴露的响应头名

## JSONP
不实用ajax请求，使用script标签请求服务器jsnp path，服务器返回调用前端callback回调方法。

## 代理
使用代理服务器，来实现跨域请求，适用于服务器不在自己手里的情况
