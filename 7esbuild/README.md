# esbuild
vite的预构建，主打打包性能优势
1.采用go语言
2.go语言多核并行，多个线程共享内存
3.不依赖第三方，从始至终使用一致的数据结构，避免无效转换
4.内存的有效利用，传统的打包工具一般会频繁的创建语法树，esbuild仅触及整个js ast 3次
  (1) 进行词法分析，解析，作用域设置和声明符号的过程
  （2）半丁符号，最小化语法，比如jsx/ts转为js
  （3）ast生成js,source map生成
   (4) 当ast数据在cpu缓存中仍然处于活跃状态时，会最大化AST数据的重用。

缺点：
1.没有提供ast操作能力，打包无法降级到es5及以下，不能兼容低版本浏览器
2.没有ts类型检测
3.默认不支持vue,等代码文件格式

vite提倡no-bundle，能做到开发时的模块按需编译，而不用先打包完再加载，但是对于第三方依赖代码node_modules，还是需要打包
webpack 先打包成一个文件，在加载
